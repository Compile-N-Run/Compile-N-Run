# C Functions

Functions are one of the most important building blocks in C programming. They allow you to organize your code into modular, reusable pieces, making your programs more structured, readable, and maintainable.

## What is a Function?

A function is a self-contained block of code that performs a specific task. Think of it as a mini-program within your main program that you can call whenever you need to perform that particular task.

```c
return_type function_name(parameter_list) {
    // function body
    // statements
    return value; // optional
}
```

## Why Use Functions?

Functions provide several advantages:

- **Code Reusability**: Write once, use many times
- **Modularity**: Break complex problems into smaller, manageable tasks
- **Abstraction**: Hide implementation details
- **Easier Debugging**: Isolate and fix issues in specific functions
- **Collaboration**: Different team members can work on different functions

## Function Declaration and Definition

In C, you typically declare a function before you use it, and define it later.

### Function Declaration (Prototype)

```c
return_type function_name(parameter_list);
```

### Function Definition

```c
return_type function_name(parameter_list) {
    // function body
    return value; // if return_type is not void
}
```

## Example of a Simple Function

```c
#include <stdio.h>

// Function declaration (prototype)
int add(int a, int b);

int main() {
    int result = add(5, 3);
    printf("Sum: %d\n", result);
    return 0;
}

// Function definition
int add(int a, int b) {
    return a + b;
}
```

## Return Types

A function can return a value of a specific data type:

- `int`, `float`, `char`, etc. - Returns a value of that type
- `void` - Returns nothing

```c
int getSquare(int num) {
    return num * num;
}

void printMessage() {
    printf("Hello, World!\n");
    // No return statement needed
}
```

## Function Parameters

Parameters are variables that receive values when the function is called:

```c
void greet(char name[]) {
    printf("Hello, %s!\n", name);
}

int main() {
    greet("John");  // Passing "John" as an argument
    return 0;
}
```

### Parameter vs. Argument

- **Parameter**: The variable in the function declaration
- **Argument**: The actual value passed to the function

## Void Parameters

A function that takes no parameters is declared with `void` or empty parentheses:

```c
void sayHello(void) {
    printf("Hello!\n");
}
```

## Function Call

You call a function by writing its name followed by parentheses containing any arguments:

```c
int main() {
    int x = 5, y = 3;
    int result;
    
    result = add(x, y);  // Function call
    
    return 0;
}
```

## Function Scope

Variables declared inside a function are local to that function and cannot be accessed outside:

```c
void someFunction() {
    int localVar = 10;  // Local variable
    printf("%d\n", localVar);
}

int main() {
    // printf("%d\n", localVar);  // Error: localVar not accessible here
    return 0;
}
```

## Function Calls in Expressions

Function calls can be part of larger expressions:

```c
int result = add(5, 3) * subtract(10, 4);
```

## Library Functions vs. User-defined Functions

- **Library Functions**: Pre-defined in C standard library (e.g., `printf()`, `scanf()`)
- **User-defined Functions**: Created by programmers for specific tasks

## Function Prototypes

It's good practice to declare all functions at the beginning of your program:

```c
#include <stdio.h>

// Function prototypes
int add(int, int);
float average(int, int);
void printMessage(void);

int main() {
    // Function calls
    return 0;
}

// Function definitions follow
```

## Best Practices for Functions

1. **Keep functions short and focused** on a single task
2. **Use meaningful function names** that describe what they do
3. **Document your functions** with comments explaining their purpose
4. **Validate input parameters** to ensure proper function behavior
5. **Consider error handling** for robust code

## Summary

Functions are essential for writing clean, modular, and maintainable C code. They help break down complex problems into smaller, manageable pieces, promote code reuse, and make programs easier to understand and debug.
