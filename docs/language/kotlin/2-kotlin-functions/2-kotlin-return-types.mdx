# Kotlin Return Types

## Introduction

When creating functions in Kotlin, one of the most important aspects to understand is the **return type** - the type of value a function produces after executing its code. Return types are a fundamental part of function signatures and help create predictable, type-safe code. In this tutorial, we'll explore how Kotlin handles function return types, from simple values to more complex scenarios.

## Basic Return Types

Every function in Kotlin has a return type, even if it doesn't explicitly return a value. Let's start with some basic examples:

### Returning Primitive Values

```kotlin
// Function that returns an Int
fun add(a: Int, b: Int): Int {
    return a + b
}

// Usage
val sum = add(5, 3) // sum = 8
```

In this example, the function signature `fun add(a: Int, b: Int): Int` specifies that the function takes two integers as parameters and returns an integer.

### Returning Strings and Other Objects

```kotlin
// Function that returns a String
fun greet(name: String): String {
    return "Hello, $name!"
}

// Usage
val greeting = greet("Kotlin") // greeting = "Hello, Kotlin!"
```

### Multiple Return Statements

A function can have multiple return statements. Kotlin will execute the first return statement it encounters:

```kotlin
fun getAbsoluteValue(number: Int): Int {
    if (number < 0) {
        return -number
    }
    return number
}

// Usage
println(getAbsoluteValue(-10)) // Output: 10
println(getAbsoluteValue(5))   // Output: 5
```

## Unit Return Type

In Kotlin, when a function doesn't return any meaningful value, it has a return type of `Unit`. This is similar to `void` in Java, but `Unit` is an actual type in Kotlin.

```kotlin
// Explicitly declaring Unit return type
fun printMessage(message: String): Unit {
    println(message)
}

// Unit return type can be omitted
fun printInfo(info: String) {
    println(info)
}

// Both functions are called the same way
printMessage("Hello") // Output: Hello
printInfo("World")    // Output: World
```

## Implicit Return Types with Type Inference

Kotlin can infer the return type of a function, which means you don't always have to specify it:

```kotlin
// Return type is inferred as Int
fun multiply(a: Int, b: Int) = a * b

// Usage
val result = multiply(4, 5) // result = 20
```

When using the single-expression function syntax (with `=`), Kotlin automatically determines the return type from the expression.

## Nullable Return Types

Kotlin's type system distinguishes between nullable and non-nullable types. You can specify a nullable return type using the `?` character:

```kotlin
// Function that might return null
fun findUser(id: Int): User? {
    // Search for user in database
    return if (userExists(id)) {
        User(id, "User $id")
    } else {
        null
    }
}

// Usage
val user = findUser(1)
if (user != null) {
    println("Found user: ${user.name}")
} else {
    println("User not found")
}
```

## Returning Collections

Functions can return collections like lists, sets, or maps:

```kotlin
// Returning a List
fun getFirstFiveEvenNumbers(): List<Int> {
    return listOf(2, 4, 6, 8, 10)
}

// Returning a Map
fun createUserMap(vararg users: User): Map<Int, User> {
    val userMap = mutableMapOf<Int, User>()
    for (user in users) {
        userMap[user.id] = user
    }
    return userMap
}

// Usage
val evenNumbers = getFirstFiveEvenNumbers()
println(evenNumbers) // Output: [2, 4, 6, 8, 10]
```

## Advanced Return Types

### Returning Functions (Higher-Order Functions)

Kotlin allows functions to return other functions, known as higher-order functions:

```kotlin
// Function that returns another function
fun createMultiplier(factor: Int): (Int) -> Int {
    return { number -> number * factor }
}

// Usage
val double = createMultiplier(2)
val triple = createMultiplier(3)
println(double(5)) // Output: 10
println(triple(5)) // Output: 15
```

### Returning Generics

You can create functions that work with generic types:

```kotlin
// Function with generic return type
fun <T> List<T>.getFirstOrNull(): T? {
    return if (this.isNotEmpty()) this[0] else null
}

// Usage
val numbers = listOf(1, 2, 3)
val empty = emptyList<String>()
println(numbers.getFirstOrNull()) // Output: 1
println(empty.getFirstOrNull())   // Output: null
```

## Using Nothing Return Type

Kotlin has a special type called `Nothing`, which represents a value that never exists. It's used for functions that never return normally (they always throw an exception or have an infinite loop):

```kotlin
fun throwError(message: String): Nothing {
    throw IllegalArgumentException(message)
}

// Usage
try {
    throwError("Something went wrong")
} catch (e: IllegalArgumentException) {
    println("Caught exception: ${e.message}")
}
```

## Real-World Application: Data Processing

Let's look at a more practical example that demonstrates different return types in a data processing scenario:

```kotlin
data class Product(val id: Int, val name: String, val price: Double)

// Returns a List of products
fun loadProducts(): List<Product> {
    // In a real app, this might fetch from a database
    return listOf(
        Product(1, "Laptop", 999.99),
        Product(2, "Phone", 699.99),
        Product(3, "Headphones", 149.99)
    )
}

// Returns a nullable Product
fun findProductById(products: List<Product>, id: Int): Product? {
    return products.find { it.id == id }
}

// Returns a Double
fun calculateTotalPrice(products: List<Product>): Double {
    return products.sumOf { it.price }
}

// Returns a Map
fun groupProductsByPriceRange(products: List<Product>): Map<String, List<Product>> {
    return products.groupBy { product ->
        when {
            product.price < 200 -> "Budget"
            product.price < 800 -> "Mid-range"
            else -> "Premium"
        }
    }
}

fun main() {
    val inventory = loadProducts()
    
    // Find a product
    val product = findProductById(inventory, 2)
    println("Found product: ${product?.name ?: "Not found"}")
    
    // Calculate total inventory value
    val totalValue = calculateTotalPrice(inventory)
    println("Total inventory value: $${totalValue}")
    
    // Group products by price range
    val groupedProducts = groupProductsByPriceRange(inventory)
    groupedProducts.forEach { (range, products) ->
        println("$range products: ${products.map { it.name }}")
    }
}
```

Output:
```
Found product: Phone
Total inventory value: $1849.97
Budget products: [Headphones]
Mid-range products: [Phone]
Premium products: [Laptop]
```

## Summary

In this tutorial, we covered:

- Basic return types for functions (Int, String, etc.)
- The `Unit` return type for functions that don't return a value
- Type inference for implicit return types
- Nullable return types with `?`
- Returning collections and complex data types
- Advanced return types like functions and generics
- The special `Nothing` return type
- Real-world applications of different return types

Understanding return types is crucial for writing clear, type-safe Kotlin code. They help the compiler verify your code's correctness and make it easier for other developers to understand what your functions do.
